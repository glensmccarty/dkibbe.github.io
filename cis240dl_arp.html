Address Resolution Protocol (ARP)

Introduction

Address Resolution Protocol is one of the requests ypur computer makes when it first connects to a network. ARP works at layer 2 of the OSI Network Model.

The purpose of ARP is to discover the hardware address for a known IP address such as a gateway.

Preparation

  1. Start Ubuntu 14.04 LTS
  2. If Wireshark isn’t installed use apt-get install wireshark to install it.
  3. Use ip r or route to find the gateway address.

Capture ARP Packets

Wireshark by itself doesn’t capture packets it only analyzes them. For that job you will use dumpcap which will have been installed along with Wireshark.

  1. Run dumpcap as root using the correct interface. In this case it the wireless card.

    $ sudo dumpcap -i wlan0 -w arp.pcapng

  2. From another terminal use arping to send an ARP REQUEST to the IP address of the gateway. You can limit the number of pings with the -c option.

  3. Use CTRL+C to terminate dumpcap.
  4. Use chmod to give everyone read/write permissions on the pcapng file.

Display the Packets in Wireshark

You do not need root privileges to read a capture file in Wireshark if the permissions have been set correctly.

  1. Start Wireshark
  2. Open the pcapng file
  3. Clcik the Protocol column header to order the protocols alphabetically. ARP should be at the top.
  4. You can also limit the display by entering arp in the filter text box and clicking apply.
