Network File System (NFS)

Overview

NFS has been around since the 1980s as a convenient why to share files over the network. In this exercise you will set up an NFS server and access files from a network client.

Assignment Objectives

When you have successfully completed this assignment you will be able to:

  1. Configure the Network File System server on CentOS 7.
  2. Configure the NFS client.
  3. Mount a remote share on a local directory.
  4. Edit /etc/fstab to make the mount available on boot.

Term You Should Know

/etc/exports

showmount -e
Show exported directories on localhost or a remote host.
exportfs -rv

rpc
Routines that allow C programs to make procedure calls on other machines across the network.

Preparation

In this assignment you will set up a NFS server for another team to connect to and they will do the same for your team. Both machines must be on the 1.92.168.201.0/24 network and have unique IP addresses.

In VirtualBox set networking to Bridged Adapter and make sure Connect the cable is checked.

Configure the NFS Server

Follow the instructions Chapter 4 to set up the server which should be on Fedora 7.

Configure the NFS Client

Set up the client following the instructions in Chapter 4 on either Ubuntu or Fedora. You will edit the /etc/fstab file to make the share available on boot.

Troubleshoot the Assignment

  • Make sure both virtual machines are on the same network and can ping each other.
  • Create a file so you can see that the directory is mounted properly.
  • Use nmap from the client to map open ports on the NFS server.
  • Port 2049 needs to be open on the server.
  • Make sure you are using the correct IP addresses for your NFS server and the subnet you are on.
