1 Overview

Samba provides secure, stable and fast file and print services for Linux and Windows clients using the SMB/CIFS protocol.

2 Assignment Objectives

_When you have successfully completed this assignment you will be able to:_ * Configure a Samba server on CentOS * Configure SELinux to work with Samba * Connect to the Samba server from a Windows workstation.

3 Terms You Should Know

/etc/samba/smb.conf
The Samba configuration file.
smb
Server Message Block daemon
nmb
NetBIOS Message Block daemon

4 Preparation

Use a CentOS virtual Machine on the 192.168.201.0/24 network.

5 Configure Samba

5.1 The Comment Characters

_Samba configuration file uses two types of comments._
# (hash)
Comments and text
; (semicolon)
Options
_Be sure to uncomment options that you change._

5.2 NetBIOS

1. Set **netbios name** to a unique name. 2. Set **interfaces** 3. Set **hosts** 4. Set **security** _Make sure the interface you enter is your active interface!_

6 Testing the Configuration

Use `testparm` to check the configuration. If any of you changes don't show up make sure you uncommented (removed the semi-colon) the line. 1. Global settings 2. Network-related 3. Logging 4. Standalone 5. Domain Controller 6. Browse Control 7. Name Resolution 8. Printer options 9. Filesystem options Restart smb after making changes.

7 Add a Samba User

1. Add a Samba user. The account must already exist. The Samba password can be different.

8 SELinux

SELinux needs to be configured for Samba. Follow the instructions in the **smb.conf** to configure SELinux. The `setsebool` command will take a while.

9 Firewall

Use `system-config-firewall-tui` to open ports for Samba. ![https://wm.sdf.org/gallery/albums/userpics/10081/firewall_samba.png](https://wm.sdf.org/gallery/albums/userpics/10081/firewall_samba.png)

10 On Your Own

SWAT is the browser-based configuration tool for Samba. See if you can set it up.
